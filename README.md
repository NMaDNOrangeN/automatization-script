# Лаунчер автоматизированной сборки (Build Automatization-Script Launcher)

Удобный графический лаунчер для запуска коллекции портативных инструментов, написанный на Python/Tkinter. Позволяет управлять списком программ (GUI и консольных), запускать их последовательно или параллельно, вести журнал запусков и формировать отчёты.  

## 📋 Возможности

- **Современный GUI** с вкладками «Запуск», «Редактор», «Отчёт».
- **Два режима запуска:**
  - *Последовательно* – каждая программа запускается после завершения предыдущей.
  - *Параллельно* – все выбранные программы стартуют одновременно в фоновых процессах.
- **Поддержка типов инструментов:**
  - **GUI** – запускаются скрыто, вывод сохраняется в лог-файл.
  - **Консольные** – открываются в интерактивном окне `cmd` в папке программы, либо с произвольными аргументами (в фоновом режиме с сохранением лога).
- **Редактор инструментов:**
  - Добавление, изменение, удаление записей.
  - Задание типа (`gui` / `console`) и относительного/абсолютного пути к исполняемому файлу.
- **Корневой путь** – можно указать общую базовую папку, тогда пути инструментов будут разрешаться относительно неё (удобно для переносных сборок).
- **Логирование** – каждый запуск записывается в `reports/launch_log.csv` (дата, время, инструмент, код возврата, путь к логу). Запись потокобезопасна.
- **Отчёты** – автоматическая генерация текстового отчёта со статистикой и выдержками из логов.
- **Портативность** – все данные (конфиги, логи) хранятся в папке с исполняемым файлом. Не требует установки, может работать с USB-флешки.

## 📁 Состав проекта

| Файл / Папка | Назначение |
|--------------|------------|
| `automatization-script.exe` | Основное GUI-приложение |
| `tools.cfg` | Конфигурационный файл – список инструментов |
| `root.cfg` | Корневой путь (одна строка) |
| `reports/` | Папка с логами и отчётами (создаётся автоматически) |
| `reports/launch_log.csv` | Журнал запусков |
| `reports/report.txt` | Сгенерированный отчёт |
| `reports/logs/` | Логи отдельных запусков |

При первом запуске, если файлы `tools.cfg` и `root.cfg` отсутствуют, они создаются пустыми.

## ⚙️ Требования

- **Windows** (используются вызовы командной строки и флаги создания процессов).

## 🚀 Быстрый старт

### Готовый EXE

1. Скачайте архив с `automatization-script.exe`.
2. Запустите `automatization-script.exe`.
3. Поместите в одну папку все программы или создайте свои конфигурационные файлы с разными путями.
4. На вкладке **Редактор** добавьте нужные инструменты (имя, тип, путь).
5. На вкладке **Запуск** выберите инструменты мышкой и нажмите «Последовательно» или «Параллельно».
6. При необходимости сформируйте отчёт на вкладке **Отчёт**.
