# Автоматизация запуска программ

Этот скрипт `launcher.ps1` позволяет управлять списком программ, запускать одиночные и множественные инструменты, а также формировать отчёты.

## Что делает скрипт

- загружает список программ из `tools.json` (перед началом работы создайте файл по примеру `tools_example.json` или загрузите свой первый путь через команду в пункте `4`)
- запускает выбранный инструмент по номеру
- позволяет выбрать несколько программ для массового запуска
- сохраняет историю каждого запуска в `reports/report.txt`
- создаёт отдельный отчёт при массовом запуске
- генерирует общий отчёт с информацией о доступных инструментах и истории запусков

## Запуск

1. Разрешите выполнение скриптов PowerShell:
    ```powershell
    Set-ExecutionPolicy UnRestricted
    ```
2. Откройте папку со скриптом.
3. Нажмите правой кнопкой мыши на скрипт и выберите пункт `Выполнить с помощью PowerShell`

## Меню

1. `Запуск инструмента` — выбираете один инструмент по номеру и запускаете его.
2. `Массовый запуск` — выбираете несколько инструментов, вводя номера через пробел или запятую, и запускаете только выбранные.
3. `Создать отчет` — формируется отчёт с текущим списком инструментов и историей запусков.
4. `Добавить программу` — добавляет новую программу в `tools.json`.
5. `Выход` — закрывает меню.

## Формирование отчётов

- `reports/report.txt` — записывается история каждого запуска с датой, именем инструмента и статусом.
- `reports/bulk-report-*.txt` — создаётся при массовом запуске и содержит список выбранных инструментов и результат их запуска.
- `reports/summary-report-*.txt` — создаётся при выборе `Создать отчет` и содержит список инструментов и историю запусков.

## Настройка `tools.json`

Файл содержит пары "имя": "путь к exe". Пример:

```json
{
  "Notepad": "C:\\Windows\\notepad.exe",
  "Calculator": "C:\\Windows\\System32\\calc.exe"
}
```

Чтобы добавить программу вручную, используйте пункт `4` или редактируйте `tools.json` напрямую.

## Примечания

- Пути к программам должны быть корректными и существовать на компьютере.
- `reports` создаётся автоматически, если папка отсутствует.
- Убедитесь, что PowerShell может выполнять скрипты  на вашем ПК.
