# Автоматизация сборки программ

Скрипт на стандартном языке командного интерпретатора Windows. Запускается двойным щелчком по `launcher.bat`
или командой `launcher.bat` в консоли.

## Файлы

| Файл              | Назначение                                              |
|-------------------|----------------------------------------------------------|
| `launcher.bat`    | сам скрипт-лаунчер                                       |
| `tools.cfg`       | список инструментов в формате `Название=Путь_к_exe`      |
| `root.cfg`        | корневой путь (PATH) к общей папке с программами (необязателен) |
| `launch_log.csv`  | автоматически создаётся, накапливает историю всех запусков |
| `report.txt`      | формируется по пункту меню «Сгенерировать отчёт»          |

Все файлы должны лежать в одной папке с `launcher.bat` — пути строятся
относительно `%~dp0`, поэтому папку можно перенести куда угодно.

## Функционал (главное меню)

1. **Запустить один инструмент** — выводит пронумерованный список из
   `tools.cfg`, пользователь вводит номер, скрипт запускает программу
   через `start /wait` и ждёт её завершения, после чего пишет строку
   в `launch_log.csv` (дата, время, название, код завершения).

2. **Массовый запуск инструментов** — тот же список, можно ввести
   несколько номеров через запятую (`1,3,4`), затем выбрать режим:
   - **последовательно** — как раньше, каждый инструмент ждёт закрытия
     предыдущего перед запуском следующего;
   - **параллельно** — все выбранные инструменты запускаются сразу,
     каждый в своём отдельном окне. Технически: скрипт запускает
     сам себя ещё раз в служебном режиме `__worker__` для каждого
     инструмента (через `start` без `/wait`), и уже этот отдельный
     процесс ждёт завершения конкретной программы и сам пишет строку
     в `launch_log.csv`. Запись в лог защищена простой блокировкой на
     основе `mkdir` (атомарная операция в файловой системе), чтобы
     параллельные процессы не портили друг другу строки при
     одновременной записи.

3. **Сгенерировать отчёт** — не запускает никакие программы, а читает уже
   накопленный `launch_log.csv` и формирует `report.txt`: сводку
   (всего запусков / успешных / с ошибкой) и таблицу со всеми записями,
   выровненную по столбцам фиксированной ширины (дата/время/инструмент/код
   завершения разделены `|`). При параллельном запуске (пункт 2) отчёт
   стоит формировать уже после того, как все открытые окна закроются.

4. **Редактор путей к инструментам** — подменю:
   - показать текущий список (с уже разрешёнными полными путями);
   - добавить новый инструмент (название + путь, абсолютный или
     относительный к корневому PATH);
   - изменить путь у существующего инструмента;
   - удалить инструмент из списка;
   - задать/изменить/отключить корневой PATH.
   Все изменения сразу сохраняются в `tools.cfg` / `root.cfg`.

## Корневой PATH (root.cfg)

Чтобы не прописывать полный путь к диску/папке в каждой строке
`tools.cfg`, можно один раз задать общий корневой путь через пункт
меню «Редактор путей → Задать/изменить корневой PATH». Он сохраняется
в `root.cfg`.

После этого в `tools.cfg` пути к инструментам можно указывать
**относительно** этой папки:

```
root.cfg:            D:\Инструменты\Программы
tools.cfg:            WinHex=winhex\winhex.exe
```

Итоговый путь, который реально запускается: `D:\Инструменты\Программы\winhex\winhex.exe`.

Правила разрешения пути для каждой строки `tools.cfg`:

- если путь уже абсолютный (начинается с `Диск:\` или `\\сервер\`) —
  используется как есть, PATH игнорируется (можно смешивать в одном
  файле абсолютные и относительные записи);
- иначе, если PATH задан — путь достраивается как `PATH\путь_из_cfg`;
- иначе (PATH не задан) — путь используется как есть (нужен полный
  путь в `tools.cfg`, как раньше).

Если перенести всю папку с программами (например, с флешки на другой
диск), достаточно один раз поменять корневой PATH — редактировать
`tools.cfg` не нужно.

## Формат tools.cfg

Простой текстовый файл, одна строка — один инструмент:

```
Название=C:\Путь\К\Программе.exe
```

Строки без `=` игнорируются, поэтому в файл можно добавлять пустые строки
или комментарии.
